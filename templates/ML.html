Technical Details

We are set out to find the next major Tech Hub in the United State to move your startup! This is how we built a machine learning algorithm to achieve this:

Data Sourcing and Cleaning

  • Gathered data from Zillow and US Census Bureau using their public web APIs
  • Cleaned and sorted data was merged using a Zip Code as the primary key
  • Loaded all data was onto a Public AWS S3 bucket for use remotely
  • ETL process completed using Pandas

    • All data was sourced from Zillow and the US Census Bureau across different features we identified as potential factors impacting start up potential. These factors included but not limited to real estate value, age, income, and education. All this data was gathered for every zipcode across the US. After the transforming of each individual dataframe, we merged both Zillow and Census tables on their zip codes to create one final table which we used for our machine learning algorithms. Our future goal is to make this data loading and manipulation automatic so that updating details in the future would be as simple as updating the files within our AWS S3 Buckets. Currently, all sourced data is from the year 2018.

Unsupervised Learning with the K-Means Model

  • Removed string data or non-prediction data variables
  • Determined optimal number of clusters through the "Elbow Method"
  • Initialized K-means model with 5 clusters

    • We imported the master.csv file containing all the state data and dropped columns containing strings (e.g. State, City, CountyName), leaving only our tech hub features and the numerical zipcode. Then, we assessed the data and removed any rows that contained any null values. Beginning the modeling process, we scaled our featured using MinMaxScaler(), fit our data and initialized the k-means model. To determine the amount of clusters that would be best for our dataset, we used the elbow method - where we choose on the graph the x-value along the elbow of the graph on what the cluster value would be. Lastly, we exported our clusters into a csv file for analysis.

Supervised Learning with the Logistical Regression Model

  • Determined cluster 4 as our definition for a tech hub
  • Defined our X, y variables using K-means clusters and split into test and training groups
  • Predicts with 95% accuracy
  • Serialized machine learning model in a .pickle file

    • We imported the cluster_predict.csv and made a new column where all rows with cluster for would equal to one and all else would equal to zero. We then defined x as the variables we wanted to use for the prediction and y as the binary cluster column. We then split our test and training model, choosing the test size to be 30% of our data. Next, we fit and trained the model and produced a training and testing score of 95%.

Database Loading and Flask Development

  • Deployed app through Heroku
  • Deserializes machine learning model
  • Requests feature values from AWS S3 bucket based on use input
  • Predicts startup fit and displays to user

    • With goals of deploying the app through Heroku, we used Flask as a web framework. With Flask native to Python, the program can deserialize the Logistical Regresion machine learning algorithm to make predictions based on the user's input. After the user enters a zip code, the data on the location is pulled from AWS and fed into the algorithm to predict the viability of a start up in the user entered location. Results are then presented to the user. We also aim to link the results to a leaflet plot to drill down into more data to help the user understand the results.
